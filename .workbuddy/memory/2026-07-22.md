## 本次修复（用户反馈：一是/二是/三是未加粗 + 编辑模式无页码）

### 根因
1. **枚举段落未加粗**：`renderBoldFirstSentence` 已支持「一是/二是/三是」开头整体加粗，
   但该逻辑仅在 `config.specialOptions.boldFirstSentence === true` 时触发，而该配置**默认 false**。
   所以 一是/二是/三是 段落永远不会加粗。
2. **编辑模式无页码**：编辑态 `.preview-page-shell` 仅 `min-height:842px`（无固定高度），
   正文 `.a4-content` 撑高后，`position:absolute; bottom:8.3%` 的「— 1 —」被推到整页最底部，
   多页文档编辑时页脚不可见。

## 本次修复（用户反馈：预览要像原仓库一样「真跟 A4 纸一样大」然后「整体缩放」）

### 思路
- 旧实现：`.a4-page` 用 `width:100%; max-width:595px; aspect-ratio` 响应式渲染（72dpi 的 595px，比真实 A4 小）。
- 新实现：「真实 A4 尺寸渲染 + 整体等比缩放」：每张纸按固定真实 A4 像素尺寸
  `794px × 1123px`（210mm×297mm @ 96dpi，与打印纸张 1:1）渲染，再用
  `transform: scale()` 等比缩放到预览区可用宽度（上限 1，不放大）。视觉上即"真 A4 纸 + 缩放到窗口"。

### 关键坑：分页度量不能被 transform 影响
- `usePagination` 通过 `getBoundingClientRect` 读页面/页边距/版头/版记高度来算分页断点。
- 可见页被 transform 缩放后 `getBoundingClientRect` 返回的是**缩放后**尺寸 → 分页错乱。
- 解法：把隐藏度量容器 `.a4-measurer` 改成**完整未缩放 A4 页**（版头+正文+版记），
  `usePagination` 读到的第一个 `.a4-page` 就是度量容器的未缩放页 → 分页始终在 794 真实坐标系计算，
  可见页仅负责缩放显示，断点天然对齐。

### 改动文件
- `src/types/documentConfig.ts`：新增 `A4_RENDER_WIDTH_PX=794`、`A4_RENDER_HEIGHT_PX=1123`、`A4_PAGE_GAP_PX=16`。
- `src/components/Preview/A4Page.css`：`.a4-page` 改为固定 `width:794px; height:1123px`（去掉 max-width/aspect-ratio）。
- `src/components/Preview/PrintPreview.css`：新增 `.preview-scale-frame`/`.preview-scale-content`；
  `.a4-measurer` 改 `width:794px`；修正 `.a4-measurer-content` 内边距为 0（避免与外层 `.a4-content` 页边距叠加导致正文变窄）。
- `src/hooks/useFitScale.ts`（新增）：根据容器可用宽度算 `scale = min(1, 可用宽/794)`，ResizeObserver 监听。
- `src/components/Preview/PrintPreview.tsx`：度量容器重构为完整未缩放 A4 页；
  `buildA4CssVars` 改用 `A4_RENDER_WIDTH_PX` 并将 `textWidth` 按 96dpi（`× 96/72`）估算真实字形宽度，保证 28 字/行仍恰好排满；
  可见页包进 `.preview-scale-content`（transform scale），外层 `.preview-scale-frame` 高度设为 `缩放后视觉高度` 预留滚动空间。
- `src/components/Preview/Preview.tsx` + `Preview.css`：编辑模式同样用真实 A4 + `.preview-scale-*` 缩放，与预览模式视觉一致；
  `getHeaderOrgFontSize` 基数由 595 改为 `A4_RENDER_WIDTH_PX`；`.preview-page-shell` 固定 794×1123；`.preview-scroll` 加 `overflow-x:hidden`。
