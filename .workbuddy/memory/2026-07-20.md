## 后续 3：一是/二是/三是 不加粗 + 手动回车右侧不换行

### Bug 1 — 一是/二是/三是 首句不被加粗
- 根因：`sanitize.ts` 的 `removeMeaninglessLineBreaks` 把不以 `：`/冒号结尾的连续普通段落合并。
  `一是…。` 之后的 `二是…。`/`三是…。` 因上一行以 `。` 结尾（非冒号、非标题、非列表符）→ 被并入同一段，
  致 `boldFirstSentence`（正文段落首句加粗）只作用于合并段第一句（即「一是…」），`二是/三是` 变普通文本。
- 修复：`sanitize.ts` 新增模块级 `SUB_ITEM_RE = /^[一二三四五六七八九十]+是/`，在合并判定中加入 `isCurrentSubItem`，
  使中文数字+「是」开头的枚举子项不向前合并，各自成为独立段落。`detectNodeType` 本就把「一是」判为 PARAGRAPH（HEADING_1_RE 需顿号），
  故开启「正文段落首句加粗」后每段首句（整句）自然加粗。
- 边界：`三是…` 之后的「发言完毕…谢谢大家！」因非子项、上段以 `。` 结尾，按既有规则并入「三是」段（属原合并设计，不在本次诉求内）。

### Bug 2 — 左侧手动回车右侧不换行
- 根因链路：`text → sanitizeText → parseGongwen(sanitized)`（`App.tsx` 调用，默认 `preserveEmptyLines=false`）把空行丢弃；
  而导出链路 `parseRichGongwen` 已用 `preserveEmptyLines=true`，两侧不一致。
- 修复：
  1. `parser.ts`：`parseGongwen` 增加可选参 `{ preserveEmptyLines?: boolean }` 透传 `parseParsedLines`（默认仍 false，向后兼容）。
  2. `App.tsx`：调用改为 `parseGongwen(sanitized, { preserveEmptyLines: true })`，手动空行如实进入预览/导出。
  3. 打印预览空段落不可见：`DocumentFlow.tsx` 与 `A4Page.tsx renderA4Content` 对空 PARAGRAPH 节点此前渲染成 `<strong></strong>`（无高度）。
     改为 `node.content` 为空时渲染 `<br />`（astToStyledHtml 本就输出 `<p><br></p>`，编辑态预览已正确；仅分页预览需补）。
- 注意：`removeRedundantSpaces` 已把 3+ 连续换行压成 2，故最多 2 个空行；正文段落之间的既有空行现在也会显示（符合「如实」预期）。

## 后续 4：一是/二是/三是 仍不加粗 + 编辑空白 + 按钮错位 + 预览多出空白页

### 用户反馈
1. 「一是、二是、三是那两句话还是没有加粗」。
2. 点击「编辑」右侧变成一片空白。
3. 「编辑」按钮与「预览」标签不在一行。
4. 预览模式内容 5 页就结束，却生成第 6、7 页空白页。

### 诊断
- **加粗失效根因**：用户原文中「一是…二是…三是…」本来就写在同一行（中间无换行），所以上轮 `sanitize` 拆分方案无法生效；`boldFirstSentence` 只加粗段落首句（即「一是…」），「二是/三是」作为同段后续文本不被加粗。
- **编辑空白根因**：`Preview.tsx` 的 `useEffect` 仅依赖 `[value]`，切换回编辑视图时 contentEditable div 重新挂载但 `value` 未变，effect 不重跑，新 div 未被注入 `innerHTML`。
- **按钮错位根因**：`.preview-header-main` 用 `align-items: baseline`，标签与按钮垂直基准不一致。
- **多出空白页根因**：`PrintPreview` 的隐藏度量容器里包了 `.a4-page > .a4-content > .a4-measurer-content`，导致 `.a4-content` padding 与 `.a4-measurer-content` padding 叠加，度量内容区比实际 A4 页内容区窄 → 换行更多 → `usePagination` 算出更多页；实际页面按真实宽度只能排 5 页，但已生成的第 6、7 页 offsetY 超出内容高度，显示空白。

### 修复
1. **枚举子项加粗**：
   - `A4Page.tsx` 的 `renderBoldFirstSentence` 增强：检测文本中「一是/二是/三是…」模式（`/[一二三四五六七八九十]+是[^。]*。/g`），存在 ≥2 处时每个子项首句都包 `<span class="a4-bold-first">` 加粗，剩余文本正常。
   - `richText.ts` 的 `renderBoldFirstSentence` 同步同样逻辑（生成 HTML string）。
   - 普通段落仍保持原首句加粗行为。
2. **编辑空白**：`Preview.tsx` 的注入 effect 依赖改为 `[value, showPrintPreview]`，切回编辑态时强制同步 `innerHTML`；同时 `.preview-editor` 增加 `min-height: 200px` 防止内容区高度塌陷。
3. **按钮对齐**：`.preview-header-main` 改为 `align-items: center; gap: 8px;`，`.preview-toggle` 的 `margin-left` 从 12px 减到 4px（gap 已提供间距）。
4. **分页空白页**：对齐参考仓库结构——去掉度量容器里的 `.a4-page/.a4-content` 包装，改为 `.a4-measurer > .a4-measurer-content > renderA4Content(...)`；`.a4-measurer-content` 保留 padding 模拟实际版心，实际 A4 页由 `usePagination` 自行选取度量。这样度量内容区宽度与实际 A4 页内容区宽度一致，分页计算不再虚高。
   - 同时删除 `PrintPreview.tsx` 中不再使用的 `renderA4Header`/`renderA4FooterNote` 导入。
